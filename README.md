# Банковский консультант: RAG-система для автоматизации ответов клиентам

## Описание проекта
Проект представляет собой MVP RAG-системы (Retrieval-Augmented Generation) для банковского консультанта. Система отвечает на вопросы клиентов по кредитам, ипотеке, вкладам, требованиям к заемщикам и типовым FAQ, используя только данные из локальной базы знаний.

## Цель
Разработать RAG-пайплайн, который:
- загружает и очищает банковские документы,
- разбивает их на чанки,
- строит эмбеддинги и сохраняет их в FAISS,
- реализует несколько вариантов retrieval,
- генерирует ответы с указанием источников,
- оценивает качество поиска и ответов,
- измеряет производительность.

## Используемые технологии
- LangChain
- FAISS
- Rank-BM25
- HuggingFace Embeddings: `intfloat/multilingual-e5-large`
- LLM: `Qwen/Qwen2.5-1.5B-Instruct`
- Ragas

## Почему выбрана `intfloat/multilingual-e5-large`
Модель подходит для retrieval-задач, поддерживает русский язык и хорошо работает в сценариях, где запросы пользователя и фрагменты документов формулируются по-разному, но должны совпасть семантически. В model card также указано использование модели для retrieval и многоязычных сценариев. ([Hugging Face model card](https://huggingface.co/intfloat/multilingual-e5-large))

## Что реализовано
### 1. Подготовка данных
- загружены 5 документов по банковским продуктам;
- текст приведен к единому формату;
- каждому документу добавлены метаданные: `source_file`, `product_type`, `doc_id`.

### 2. Чанкинг
Реализованы три стратегии:
- `CharacterTextSplitter`
- `RecursiveCharacterTextSplitter`
- `NLTKTextSplitter`

В проекте для основной системы выбран `RecursiveCharacterTextSplitter`, так как он лучше сохраняет логическую структуру документов.

### 3. Векторная база и эмбеддинги
- создаются эмбеддинги документов
- строится и сохраняется индекс FAISS
- возможна повторная загрузка индекса без повторной векторизации

### 4. Retrieval
Реализованы:
- similarity search
- MMR search
- BM25
- гибридный retrieval (dense + sparse)
- metadata filtering
- контекстное сжатие

### 5. Генерация ответа
- используется системный промпт для банковского консультанта
- модель отвечает только на основе контекста
- если точного ответа нет, система сообщает об этом
- в ответ добавляются источники

### 6. Оценка качества
Для retrieval реализованы:
- HitRate@3
- MRR

Для end-to-end оценки предусмотрен модуль `ragas_eval.py`, который использует:
- Faithfulness
- Answer Relevancy
- Context Precision

## Как запустить
### 1. Создать окружение
```bash
python -m venv venv
```

### 2. Активировать окружение
**Windows:**
```bash
venv\Scripts\activate
```

**Linux / macOS:**
```bash
source venv/bin/activate
```

### 3. Установить зависимости
```bash
pip install -r requirements.txt
```

### 4. Запустить проект
Из корня проекта:
```bash
python src/main.py
```

## Что появится после запуска
В папке `results/` сохраняются:
- `chunk_stats.json`
- `retrieval_metrics.json`
- `performance_results.json`
- `ragas_results.json` (если запускалась оценка через Ragas)

## Возможные улучшения
- query rewriting вместо наивной памяти диалога
- reranking retrieved chunks
- router-agent для разделения FAQ / search / calculator
- отдельный банковский калькулятор как Tool
- более строгая валидация ответов по источнику

## Ограничения
- синтетическая база знаний мала и не отражает всех банковских сценариев
- локальная LLM может работать медленно на слабом железе
- наивная обработка истории диалога может приводить к topic leakage, если её не ограничивать
